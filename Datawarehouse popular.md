# คลังข้อมูลบนคลาวด์

1. อเมซอน เรดชิฟต์
บริการคลังข้อมูลที่สามารถปรับให้เหมาะสมกับกรณีการใช้งานเฉพาะและได้รับการจัดการอย่างเต็มรูปแบบโดยAWSเมื่อต้องวิเคราะห์ข้อมูลปริมาณมหาศาล มีรูปแบบการจัดเก็บข้อมูลแบบคอลัมน์เพื่ออำนวยความสะดวกในการค้นหาข้อมูลที่มีโครงสร้าง

คุณสมบัติหลัก:
การจัดเก็บข้อมูลแบบคอลัมน์เพื่อการค้นหาข้อมูลเชิงวิเคราะห์ที่รวดเร็ว
สถาปัตยกรรม MPP (การประมวลผลแบบขนานขนาดใหญ่)
การปรับขนาดอัตโนมัติและการคิดราคาตามการใช้งาน
การผสานรวมอย่างเป็นธรรมชาติกับระบบนิเวศของ AWS (S3, Glue, Athena)


2. Microsoft Azure Synapse Analytics
ชุดโปรแกรมคลังข้อมูล เช่นAzure Synapse Analyticsที่ใช้ระบบคลาวด์คอมพิวติ้ง ช่วยในการสร้าง ปรับใช้ และจัดการโซลูชันคลังข้อมูลด้วย ความสามารถ ด้านแมชชีนเลิร์นนิงภายในสถาปัตยกรรม

คุณสมบัติหลัก:
การประมวลผลแบบสอบถามแบบกระจาย
ระบบรักษาความปลอดภัยในตัวด้วย Azure Active Directory
การผสานรวมอย่างแน่นหนากับ Power BI
รองรับการวิเคราะห์แบบเรียลไทม์และแบบกลุ่ม


3. Google BigQuery
เครื่องมือสำหรับวิเคราะห์ข้อมูลขนาดใหญ่ (Big Data)ที่ออกแบบมาโดยเฉพาะสำหรับคลังข้อมูล (Data Warehouse) โดยไม่ต้องจัดการเซิร์ฟเวอร์ ใช้งานแบบจ่ายตามการใช้งานจริง และมีจุดเด่นคือการประมวลผลคำสั่งค้นหาที่รวดเร็วยิ่งขึ้นด้วยการรองรับSQL

คุณสมบัติหลัก:
BigQuery เป็นฐานข้อมูลอัจฉริยะที่ช่วยอำนวยความสะดวกในการประมวลผลชุดข้อมูลขนาดใหญ่เพื่อการวิเคราะห์เชิงลึก ช่วยลดความจำเป็นในการแทรกแซงของมนุษย์บนเซิร์ฟเวอร์ และสามารถเพิ่มทรัพยากรโดยอัตโนมัติหากคำขอค้นหาเกินขีดจำกัดของทรัพยากรที่มีอยู่ อีกทั้งยังช่วยอำนวยความสะดวกในการจัดเก็บข้อมูลขนาดใหญ่เพื่อการวิเคราะห์ข้อมูลขนาดใหญ่ (Big Data Analytics)
BigQuery คือโซลูชันพื้นที่จัดเก็บข้อมูลบนคลาวด์ของ Google ที่คิดค่าบริการเฉพาะเมื่อคุณใช้งานเท่านั้น ทำให้เป็นตัวเลือกที่คุ้มค่าที่สุดสำหรับการทดสอบข้อมูลปริมาณต่างๆ
บริการ Google Cloud มีฟีเจอร์เพิ่มเติม เช่น Cloud AI Platform สำหรับ BigQuery เมื่อใช้บริการ BigQuery เพื่อทำการวิเคราะห์ทางสถิติ หรือใช้ความสามารถในการวิเคราะห์ขั้นสูงเพื่อดึงข้อมูลเชิงลึกใหม่ๆ จากข้อมูล


4. เกล็ดหิมะ
คลังข้อมูลบนคลาวด์ชั้นนำที่ขึ้นชื่อเรื่องความสามารถในการปรับขนาดทรัพยากรจัดเก็บและประมวลผลได้อย่างอิสระ ทำให้สามารถบริหารจัดการต้นทุนได้อย่างยืดหยุ่นและลดความซับซ้อนในการจัดการข้อมูล

คุณสมบัติหลัก:
รองรับข้อมูลที่มีโครงสร้างและกึ่งโครงสร้าง (JSON, Parquet, Avro)
การแยกส่วนการจัดเก็บข้อมูลและการประมวลผล
การแบ่งปันข้อมูลอย่างปลอดภัย
การใช้งานระบบคลาวด์หลายระบบ (AWS, Azure, GCP)


5. ไมโครโฟกัส เวอร์ติกา
คลังข้อมูลที่มีสถาปัตยกรรมที่จำเป็นสำหรับประสิทธิภาพสูงสุดในการจัดการข้อมูลจำนวนมาก ออกแบบมาบนหลักการประมวลผลแบบขนานขนาดใหญ่ (MPP) ซึ่งทำให้สามารถประมวลผลคำสั่งค้นหาได้อย่างรวดเร็ว และเป็นเครื่องมือวิเคราะห์ที่มีประสิทธิภาพสูง

คุณสมบัติหลัก:
สถาปัตยกรรม MPP
การรับข้อมูลและการวิเคราะห์แบบเรียลไทม์
การเร่งความเร็วการค้นหาในหน่วยความจำ
การจัดการปริมาณงานขั้นสูง

การคิดราคาตามจำนวนคำขอ


# ฐานข้อมูลเชิงสัมพันธ์ที่ใช้ในคลังสินค้า45455

## โพสต์เกรสซีอาร์

ระบบจัดการฐานข้อมูลเชิงสัมพันธ์ (RDBMS) แบบโอเพนซอร์สที่มีประสิทธิภาพสูง เป็นที่รู้จักในด้านความน่าเชื่อถือและฟังก์ชันการทำงานที่ครบครัน รองรับการสืบค้นข้อมูลที่ซับซ้อนและทำงานร่วมกับเครื่องมือ BI ต่างๆ ได้เป็นอย่างดี

คุณสมบัติหลักของPostgreSQL
1. PostgreSQLเป็นระบบจัดการฐานข้อมูลเชิงสัมพันธ์ (RDBMS) แบบโอเพนซอร์สที่มีประสิทธิภาพสูง ขึ้นชื่อเรื่องความน่าเชื่อถือและฟังก์ชันการทำงานที่ครบครัน รองรับการสืบค้นข้อมูลด้วยคำสั่ง SQL ที่ซับซ้อน
2. ช่วยให้มั่นใจได้ถึงความสม่ำเสมอและความสมบูรณ์ของข้อมูลผ่านการรองรับธุรกรรมACID (Atomicity, Consistency, Isolation, Durability)
3. ประกอบด้วยคุณสมบัติต่างๆ เช่น ทริกเกอร์ สตored procedure และฟังก์ชันที่ผู้ใช้กำหนดเองสำหรับการจัดการข้อมูลขั้นสูง
4. สามารถทำงานร่วมกับเครื่องมือ BI และแพลตฟอร์มวิเคราะห์ข้อมูลต่างๆ ได้เป็นอย่างดี เพื่อการแสดงภาพข้อมูลและการรายงาน

# การจัดเก็บข้อมูลแบบอ็อบเจ็กต์ (ใช้ในระบบคลังข้อมูลสมัยใหม่)

## อเมซอน เอส3
บริการจัดเก็บข้อมูลแบบอ็อบเจ็กต์ที่ปรับขนาดได้และคุ้มค่า สำหรับจัดเก็บข้อมูลที่ไม่เป็นระเบียบจำนวนมาก เช่น รูปภาพ วิดีโอ และไฟล์บันทึก ให้ความทนทานสูง ความปลอดภัย และการเข้าถึงข้อมูลได้ง่าย

คุณสมบัติหลัก:
1. S3 นำเสนอโซลูชันที่ปรับขนาดได้และคุ้มค่าสำหรับการจัดเก็บข้อมูลที่ไม่เป็นระเบียบจำนวนมาก เช่น รูปภาพ วิดีโอ บันทึก และไฟล์สำรองข้อมูล
2. เก็บรักษาเวอร์ชันเก่าของวัตถุไว้ ทำให้คุณสามารถย้อนกลับไปยังเวอร์ชันก่อนหน้าได้หากจำเป็น
3. ระบบนี้ช่วยจัดการวงจรชีวิตของข้อมูลโดยอัตโนมัติ ด้วยการกำหนดกฎสำหรับการจัดเก็บหรือลบข้อมูลตามเกณฑ์ที่กำหนด
คลังข้อมูลองค์กร

## คลังข้อมูล IBM Db2

คุณสมบัติหลักของ IBM Db2 Warehouse

คุณสมบัติหลัก:
การจัดเตรียมและการปรับแต่งอัตโนมัติ
การอัปเดตแพทช์อัตโนมัติและการสำรองข้อมูล
การประมวลผลคิวรีที่มีการทำงานพร้อมกันสูง
